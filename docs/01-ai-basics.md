# 01 - AI 基础：什么是大模型？

> 面向 iOS 开发者的 AI 入门指南，用你熟悉的概念来理解大模型。

## 用 iOS 开发者的视角理解大模型

### 类比：大模型就像一个超级 NSDictionary

作为 iOS 开发者，你一定用过 `NSDictionary`。想象一个超大的字典：

```
输入（Key）: "今天天气"
输出（Value）: "怎么样？" / "不错" / "很热" ...
```

大模型本质上就是一个**超级复杂的映射函数**：给定输入文本，输出最可能的下一个词。

但它不是简单的 key-value 查找，而是通过**数十亿个参数**（类似权重）来学习语言规律。

### 关键概念速查表

| 概念 | iOS 类比 | 解释 |
|------|----------|------|
| **模型 (Model)** | `.mlmodel` 文件 | 存储了学到的知识的文件，包含大量参数 |
| **参数 (Parameters)** | 权重矩阵中的数字 | 模型学到的知识，以数字形式存储 |
| **推理 (Inference)** | `model.prediction(from: input)` | 给模型输入，让它输出结果 |
| **Token** | 类似字符/词的切片 | 模型处理文本的最小单位，"你好世界" ≈ 2-4 个 token |
| **上下文窗口 (Context)** | 类似 buffer size | 模型一次能"看到"的最大文本长度 |
| **量化 (Quantization)** | 类似图片压缩 | 用更少的位数表示参数，减小模型体积 |

## 模型的"大"到底有多大？

```
模型参数量对比：

GPT-4:        ~1.8 万亿参数  ← 只能在云端运行
Llama 3 70B:  700 亿参数     ← 需要专业 GPU 服务器
Llama 3.2 3B: 30 亿参数      ← 可以在 iPhone 上运行！🎯
Gemma 2B:     20 亿参数      ← 更小，iPhone 友好
Phi-3-mini:   38 亿参数      ← 微软出品，适合端侧

参数量 → 文件大小（粗略估算）：
1B 参数 ≈ 2GB (FP16) ≈ 0.5GB (INT4 量化)
```

### iPhone 内存限制

```
iPhone 15 Pro:  8GB RAM  → 可运行 1-3B 量化模型
iPhone 16 Pro:  8GB RAM  → 同上，但 NPU 更强
iPad Pro M4:   16GB RAM  → 可运行 7B 量化模型
```

> **重点**：端侧模型的关键限制是**内存**，不是算力。iPhone 的 NPU（Neural Engine）算力很强，
> 但可用内存决定了能跑多大的模型。

## 大模型是怎么"说话"的？

### 自回归生成（Auto-regressive Generation）

```
用户输入: "写一首关于iOS的诗"

模型的工作过程（逐字生成）：
Step 1: "写一首关于iOS的诗" → "代"     (预测下一个 token)
Step 2: "写一首关于iOS的诗代" → "码"    (基于已有文本预测)
Step 3: "写一首关于iOS的诗代码" → "之"
Step 4: "写一首关于iOS的诗代码之" → "美"
...

就像打字时的自动补全，但更智能、更连贯。
```

### 两个关键阶段

```
┌─────────────────┐     ┌─────────────────────────────────┐
│   Prefill 阶段   │────▶│        Decode 阶段              │
│  处理输入 prompt  │     │  逐个生成 token（你看到的打字效果）│
│  一次性并行计算   │     │  串行，一次生成一个 token        │
│  耗时较短        │     │  耗时较长（主要瓶颈）            │
└─────────────────┘     └─────────────────────────────────┘

iOS 开发者关注点：
- Prefill 速度影响 "首字延迟"（用户等待第一个字出现的时间）
- Decode 速度影响 "生成速度"（每秒能输出多少个字）
- 两个指标都是 Benchmark 的核心
```

## 对 iOS 开发者的意义

### 为什么要关注端侧 AI？

1. **隐私** — 数据不出设备，符合 Apple 隐私理念
2. **离线** — 无需网络即可使用
3. **低延迟** — 省去网络往返时间
4. **免费** — 不需要付 API 费用
5. **Apple Intelligence** — Apple 正大力推进端侧 AI

### 端侧 AI 的现实局限

1. **能力有限** — 小模型不如大模型聪明
2. **内存紧张** — iPhone 内存远少于服务器
3. **发热** — 持续推理会导致手机发热降频
4. **模型下载** — 用户需要下载几百MB到几GB的模型文件
